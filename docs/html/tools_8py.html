int GPIO_AL.tools.argToLine ( Union  pinArg[int, str])

Convert a GPIO_AL pin argument to an RB Pi GPIO chip "line" or "offset" numbers (2 .

. 27). The argument can be an integer, in which case it is assumed that this integer represents a GPIO header pin number (1 .. 40). The argument can also be a string, in which case it is interpreted as a line number if it starts with the string 'GPIO'; otherwise it will be interpreted as representing an integer pin number.

◆ gpioChipPath()

str GPIO_AL.tools.gpioChipPath ( int   line)

Obtain the path to a GPIO chip that contains a given line.

The function will ignore lines that are used. Note that the argument is a line or offset number NOT a GPIO header pin number! (

See also
argToLine())

As long as the GPIO line names all start with GPIO and are followed by the line number, this function does not need to be changed if the architecture changes.

Raspberry Pi versions before RB Pi 5 used the GPIO chip /dev/gpiochip0 to address all GPIOI lines (offsets). Early versions of RB Pi 5 used /dev/gpiochip4, which later was reversed to gpiochip0 while gpiochip4 still existed as a symbolic link to gpiochip0. To stay above all this mess, and even accomodate an architecture where GPIO lines are split across several GPIO chips, this code attempts to find the right chip for a given line programmatically. This approach has the additional advantage, that it will also work on all Linux machines (e.g. also on a Jetson Nano running gpiod v2.2 and above).

Parameters
lineGPIO line (offset) to search for in the GPIO chips
Returns
path to GPIO chip containing that (unused) line as a string
